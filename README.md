# 대학교 수강신청 시스템

이 저장소는 대학교 수강신청 시스템 백엔드를 구현하는 작업 공간이다.  
실제 애플리케이션 코드는 `src/schoolSystem`에 두고, 요구사항 해석과 작업 흔적은 루트 문서에 분리해 관리한다.

## 폴더 구조

- `src/schoolSystem`
  - Spring Boot 기반 백엔드 프로젝트
  - 수강신청, 조회, 헬스체크, 동시성 검증 관련 코드와 테스트를 포함한다
- `docs`
  - 요구사항 해석, 문제 분석, API 정리, 체크리스트 등 설계 판단을 기록한다
- `prompts`
  - Codex에 어떤 요청을 했는지, 어떤 식으로 작업을 진행했는지 프롬프트 이력을 남긴다
- `AGENTS.md`
  - Codex가 이 저장소에서 작업할 때 따라야 하는 우선순위와 작업 원칙을 정의한다
- `PROBLEM.md`
  - 과제 원문 또는 구현 기준이 되는 문제 설명 문서다

## Codex 활용 방식

이 저장소는 단순히 코드를 생성하는 용도보다, 구현 과정 자체를 구조화하는 데 Codex를 활용하는 방식으로 정리했다.

- 먼저 `AGENTS.md`에 작업 원칙을 정의해 Codex가 구현 위치와 우선순위를 일관되게 따르도록 했다
- 요구사항 해석이나 설계 결정은 코드에 섞지 않고 `docs/`에 남기도록 유도했다
- 실제 프롬프트와 작업 흐름은 `prompts/`에 기록해, 어떤 지시로 어떤 결과를 만들었는지 추적할 수 있게 했다
- 핵심 요구사항인 동시성 제어, 학점 제한, 시간표 충돌 검증처럼 실패 비용이 큰 축을 중심으로 Codex 작업 범위를 좁혔다
- 실행 가능한 서버, 헬스체크, 테스트 가능한 구조를 먼저 확보한 뒤 기능을 확장하는 순서로 활용했다

## 실행

```bash
cd src/schoolSystem
./gradlew bootRun
```

기본 포트는 `8080`이며, 헬스체크는 아래로 확인할 수 있다.

```bash
curl http://localhost:8080/health
```
